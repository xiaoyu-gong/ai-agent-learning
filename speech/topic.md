# topic.md — 写什么

## 演讲主题
XM AI 智能体开发教学项目汇报演讲

## 信息来源
- **唯一素材**：`report.md`（项目报告全文）
- 所有数据、结论、技术细节必须源自 report.md，不得凭空编造
- 不得引用 report.md 中未出现的第三方资料

## 覆盖范围
- **必须覆盖**：项目全部 7 个 Practice 模块的整体介绍（Practice 01–07）
- **必须提及**：零第三方依赖、Skill 热加载系统、链式工具调用三大核心创新点
- **必须包含**：至少 2 组测试数据（压缩测试、链式调用测试）
- **可简要提及**：技术选型理由（为什么用 http.client、Qwen2.5、AnythingLLM）

## 禁区
| 禁止行为 | 说明 |
|---------|------|
| 深入单个模块代码细节 | 3 分钟演讲没有时间展开代码级讨论 |
| 偏离 XM 项目主题 | 不得谈论 AI Agent 行业趋势、其他框架对比等泛化话题 |
| 虚构不存在的数据 | 所有数据必须能在 report.md 中找到出处 |
| 评价式而非陈述式 | "做得很好"类主观评价改为"实现了 X"类客观陈述 |
| 遗漏测试验证章节 | 正式学术汇报必须具备数据验证环节 |

## 演讲者视角
- **第一人称**（"我"/"本项目"）
- **身份**：项目作者本人
- **立场**：客观陈述设计决策与实现结果，不夸大、不回避不足
