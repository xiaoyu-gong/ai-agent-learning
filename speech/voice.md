# voice.md — 怎么写

## 语气定位：正式学术风

### 整体基调
- 客观、克制、精准
- 陈述事实优先于抒发感受
- 逻辑驱动而非情感驱动

### 人称规则
| 场景 | 使用 |
|------|------|
| 描述设计决策 | "我选择……" / "本项目采用……" |
| 描述实现过程 | "我实现了……" / "代码中通过……" |
| 描述测试结果 | "测试表明……" / "数据显见……" |
| 提及听众 | "各位老师、同学"（仅限开场一次） |

### 禁用词汇表
| 禁用 | 原因 | 替换 |
|------|------|------|
| "综上所述" | 机械套路 | 直接给出结论 |
| "众所周知" | 空洞断言 | 不出现 |
| "非常"/"极其" | 主观修饰 | 用数据替代 |
| "很棒"/"优秀" | 自我评价 | 改为客观陈述 |
| "众所周知" | 假设听众已知 | 简要说明 |
| "我想说的是" | 冗余引导 | 直接陈述 |
| "在这里" | 口语化赘词 | 删除或替换 |

### 句式要求
- **短句为主**：单句不超过 40 字，适合口头表达
- **避免嵌套从句超过 2 层**
- **每段开头用主题句**，段内展开
- **数据陈述句式**："X 项测试中，Y 项通过，通过率 Z%"

### 节奏控制
- 语速 300 字/分钟，总稿 ≤900 字
- 关键数据点前后预留 0.5 秒停顿感（用短句/换行暗示）
- 避免连续 3 句以上同等长度（产生单调感）
