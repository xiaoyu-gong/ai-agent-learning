# check.md — 写完后查什么

## 一、硬性约束检查（不合格即重写）

| 检查项 | 标准 | 方法 |
|--------|------|------|
| 总字数 | ≤900 字 | 字数统计工具 |
| 段落数 | 5 段（开场白、背景、方案、验证、总结） | 人工计数 |
| 素材来源 | 100% 源自 report.md | 逐句对照 report.md |
| 第一人称一致性 | 全文使用"我"/"本项目"，不得出现"我们"/"作者" | 搜索关键词 |

## 二、内容准确性检查

| 检查项 | 标准 |
|--------|------|
| 模块数量 | 7 个（Practice 01–07），不得漏写或多写 |
| 痛点数量 | 2-3 个核心痛点，与 report.md 1.1 一致 |
| 测试数据 | 压缩率 71.5%、链式调用 3/3 通过，与 report.md 4.2 一致 |
| 创新点 | 零依赖、Skill热加载、链式调用——三项必须全部提及 |
| 不足 | 至少提及 1 个，不得回避 |

## 三、风格检查

| 检查项 | 通过标准 |
|--------|---------|
| 禁用词扫描 | 不含"综上所述""众所周知""非常""极其"等 voice.md 中的禁用词 |
| 单句长度 | 任意单句不超过 40 字（超出则拆分） |
| 嵌套从句 | 不超过 2 层嵌套 |
| 客观性 | 不含"很棒""优秀""非常好"等主观评价词 |
| 学术性 | 数据均有出处，结论均有依据 |

## 四、演讲适读性检查

| 检查项 | 方法 |
|--------|------|
| 朗读测试 | 出声朗读全文，检查是否有拗口长句、连续同等长度句子 |
| 时间测试 | 按 300 字/分钟语速朗读，计时是否在 2:45-3:15 范围内 |
| 重音点 | 标注关键数据处（如 "71.5%"、"3/3 通过"），确保重读 |
| 结尾力度 | 最后一句是否干脆有力（不得拖沓、不得突然中止） |

## 五、禁止事项

- ❌ 不得添加 report.md 中没有的新数据
- ❌ 不得评价第三方框架（"LangChain 很复杂"等）
- ❌ 不得使用反问句（"那么我是怎么解决的呢？"）
- ❌ 不得出现"谢谢聆听"（"聆听"为敬词，演讲者自称不当）
- ❌ PPT 式分点列表（演讲稿是连续文本，非幻灯片提纲）
