# structure.md — 写成什么样

## 总约束
- **总字数**：≤900 字
- **总时长**：≤3 分钟（按 300 字/分钟）
- **段落数**：5 段（对应 5 个时间块）

## 五段式结构

### 第一段：开场白（≈100 字 / 20 秒）
| 要素 | 内容 |
|------|------|
| 问候 | "各位老师、同学，大家好"（仅此一次） |
| 身份 | "我是本项目的作者" |
| 主题 | 一句话概括项目："一个基于 Python 标准库、零第三方依赖的 AI Agent 渐进式教学项目" |
| 路线图预告 | 用一句话预告演讲结构："我将从项目背景、技术方案、测试验证三个方面进行汇报" |

### 第二段：背景与痛点（≈200 字 / 40 秒）
| 要素 | 内容 |
|------|------|
| 痛点列举 | 从 report.md 1.1 节提取 4 个痛点，简述 2-3 个核心痛点 |
| 痛点聚焦 | "框架黑箱"和"依赖爆炸"作为主讲痛点 |
| 目标陈述 | "基于此，我设计了 7 个渐进式实践模块" |

### 第三段：核心方案（≈300 字 / 60 秒）
| 要素 | 内容 |
|------|------|
| 技术选型 | 为何选 http.client 而非 requests（1-2 句） |
| 7 模块简介 | 按演进路线简述：HTTP → 流式 → Function Calling → 压缩 → 日志 → Skill → 链式调用 |
| Skill 系统亮点 | YAML front matter + 热加载机制（2-3 句） |
| 链式调用亮点 | ChainedCallContext + 双格式决策（2-3 句） |

### 第四段：测试验证（≈200 字 / 40 秒）
| 要素 | 内容 |
|------|------|
| 压缩测试数据 | report.md 4.2 节：4521字符→1287字符，压缩率71.5% |
| 链式调用数据 | 3 个场景全部通过（文件搜索、多文件运算、网页处理） |
| 性能数据 | 选取1-2项关键指标（如 token 生成速度 16.3 tokens/s） |

### 第五段：总结与展望（≈100 字 / 20 秒）
| 要素 | 内容 |
|------|------|
| 成果回顾 | "本项目实现了零依赖教学闭环、Skill 热加载系统和链式调用引擎三项核心成果" |
| 不足 | 简要提及 1-2 个不足（如本地部署硬件要求、提示词依赖） |
| 未来方向 | 提及 1-2 个方向（如集成更强模型、Web 可视化） |
| 收尾 | "以上是我的汇报，请各位老师批评指正" |

## 时间轴校验
```
0:00 ─── 0:20 ─── 1:00 ─── 2:00 ─── 2:40 ─── 3:00
 开场白    背景痛点    核心方案    测试验证    总结展望
 (100字)   (200字)    (300字)    (200字)    (100字)
```

## 段落过渡规则
- 段与段之间用逻辑连接，不使用"接下来"/"然后"等口语过渡词
- 用内容本身的逻辑递进完成过渡
- 示例：痛点说完自然过渡到"针对这些痛点，我设计了……"
