# 如何工作

根据用户需求询问用户问题，直到掌握了足够多的信息后，创作撰写文档的四个约束规则文件：

| 文件             | 一句话定位      | 解决什么问题          |
| -------------- | ---------- | --------------- |
| `topic.md`     | **写什么**    | 防止跑题、防止 AI 自说自话 |
| `voice.md`     | **怎么写**    | 设定语气，防止套路腔、防止"综上所述"  |
| `structure.md` | **写成什么样**  | 防止结构混乱、防止流水账    |
| `check.md`     | **写完后查什么** | 防止空话、防止没个人见解，防止学术造假，防止学术不端    |
