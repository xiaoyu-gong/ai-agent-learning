# 文档结构与提纲规范 (Structure Blueprint)

## 3.1 元结构原则
所有文档遵循"倒金字塔+模块化"原则：
- 顶部20%：核心结论/主旨全景（让读者30秒内抓住重点）
- 中部60%：分层论证/展开（每部分可独立阅读）
- 底部20%：总结升华/行动号召（回扣开头，形成闭环）

## 3.2 通用章节骨架
```text
[封面/标题区]
  └─ 标题公式：核心对象 + 关键动作 + 价值指向（限20字内）

[摘要/导语区]（占全文5-10%）
  └─ 三要素：背景（为何写）+ 核心发现（是什么）+ 结构预告（怎么读）

[主体论证区]（占全文60-70%）
  └─ 采用"MECE原则"：相互独立，完全穷尽
  └─ 每章遵循"论点→论据→论证→小结"四步循环

[边界/局限区]（占全文5%，可选）
  └─ 主动声明本文未覆盖的内容或存在的局限

[结论/行动区]（占全文10-15%）
  └─ 不引入新论据，仅整合前文并指向未来
```

## 3.3 题材定制模板

### 模板A：软件工程报告
```text
1. 项目背景与问题定义（痛点数据化）
2. 技术方案选型（对比维度：性能/成本/维护性）
3. 架构设计与实现（分层图+关键代码逻辑）
4. 测试验证与数据（基准测试+压力测试）
5. 部署运维与监控（CI/CD流程+告警策略）
6. 总结与演进路线（已解决/待优化/下一步）
```

### 模板B：读书心得
```text
1. 书籍与作者定位（一句话定义该书在知识谱系中的位置）
2. 核心论点梳理（作者的原意，非我的感想）
3. 关键概念深读（选取2-3个核心概念，结合原文细读）
4. 个人经验勾连（具体场景，拒绝泛泛而谈）
5. 批判性思考（同意什么/质疑什么/补充什么）
6. 行动转化（这本书如何改变了我接下来的一个具体行为）
```

### 模板C：入党申请书
```text
1. 入党动机（真实触发点，拒绝模板化表述）
2. 思想成长历程（理论学习的具体书目与心得节点）
3. 实践服务经历（时间/地点/事件/反思，STAR法则）
4. 自我剖析与不足（具体缺点+改进计划，忌"过于追求完美"）
5. 未来承诺（可量化的行动指标，而非口号）
```

## 3.4 段落微观结构
- **起承转合标记**：每段开头使用逻辑路标词（"具体而言"、"反观之"、"值得注意的是"）
- **信息密度**：每段不超过5个信息点，超过则拆分
- **过渡规则**：段与段之间必须有显性连接词或隐性逻辑链（禁止硬跳转）

## 3.5 长度控制公式
- 总字数 = 基础值 × 复杂度系数
  - 基础值：短篇(1500) / 中篇(3000) / 长篇(5000)
  - 复杂度系数：单一主题(1.0) / 对比分析(1.3) / 系统性综述(1.6)
