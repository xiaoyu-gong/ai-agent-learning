# 主题锚定规范 (Topic Anchoring)

## 1.1 核心主题声明
- **主命题**：[在此填入文档的核心命题，一句话概括]
- **边界范围**：
  - 必须涵盖：[列出3-5个必须涉及的关键维度]
  - 明确排除：[列出2-3个易跑偏的相关话题，明确声明不讨论]
- **目标读者**：[描述预期读者画像，影响内容深度与术语使用]

## 1.2 偏题检测机制
生成过程中，每完成一个章节需自检：
- [ ] 本节内容是否直接支撑主命题？
- [ ] 若删除本节，主命题是否仍然完整？（如答案是"是"，则本节可能冗余）
- [ ] 本节是否引入了边界范围外的新话题？

## 1.3 题材适配示例
| 题材类型 | 主命题示例 | 边界排除示例 |
|---------|-----------|------------|
| 软件工程报告 | 本系统如何通过微服务架构解决高并发场景下的性能瓶颈 | 不讨论前端UI设计细节、不对比其他公司商业产品 |
| 读书心得 | 《xxx》中"苦难美学"对当代青年精神困境的启示 | 不介绍作者生平年表、不讨论该书出版市场反响 |
| 入党申请书 | 个人在思想/实践/学习三维度向党组织靠拢的历程与决心 | 不罗列所有获奖证书、不抱怨社会负面现象 |

## 1.4 纠偏触发词
当内容偏离时，使用以下句式强制回归：
"回到核心命题：[重复主命题]，我们需要聚焦于..."
